00
隐 私 和 防 外 泄 总 原 则
Privacy First
必须遵守

本文档禁止写入任何真实凭证。不得把 VLESS/VMess/Trojan/SS 链接、二维码、UUID、Reality PublicKey、ShortID、订阅 URL、面板 URL、面板路径、管理员账号、邮箱、Server 酱/Telegram/Slack token 写进 HTML、截图、群聊、GitHub、Notion、飞书公开文档或任何可被转发的位置。

一人一证
Per-member UUID

每位用户使用独立成员凭证，不共享 Owner 凭证。成员离职、设备丢失或疑似外泄时，只撤销该成员，不影响其他人。

文档无密
No secrets in handbook

HTML 只放占位符和 SOP。真实链接由管理员通过密码管理器、端到端加密私聊或一次性安全通道单独发放。

最小可见
Least disclosure

用户只需要自己的链接，不需要知道面板地址、其他成员列表、服务端配置、私钥、日志位置或管理员脚本。

可追踪可回收
Traceable revoke

凭证备注使用不含真实姓名的内部代号，例如 member-a-laptop。外泄时按代号定位并撤销。

用户使用守则

允许避免禁止

01
配 置 变 量 和 占 位 符
Template Variables

管理员发放给每位成员的真实参数必须单独传递。本文档中的 {...} 都是占位符，不代表真实值。

变量含义是否可给用户备注
{VPN_NAME}节点显示名可以例如 ArchIToken-VPN / Team VPN，不含真实机房细节
{MEMBER_CONFIG_URL}成员专属导入链接仅给本人不要写入 HTML；单独加密发送
{MEMBER_QR}成员专属二维码仅给本人二维码等价于明文凭证
{EXPECTED_EXIT_IP}验证用出口 IP（主/备节点之一）按需可只发给需要自检的用户；用户版默认写占位符
{PRIMARY_SERVER_IP}主节点服务器地址谨慎手动配置时才需要；默认不公开
{SECONDARY_SERVER_IP}备用节点服务器地址谨慎手动配置时才需要；默认不公开
{PANEL_URL}管理面板地址禁止管理员专用
{PRIVATE_KEY}服务端私钥禁止不得离开服务端
推荐发放方式

管理员给每位用户发送一条成员专属链接，并附上本 HTML。链接不要放在 HTML 内；HTML 可以广泛发，链接只能给本人。

剪贴板优先

跨平台导入时，剪贴板导入比扫码更可靠。二维码有时会丢 query 参数，导致 Reality、Flow、加密字段缺失。

03
一 键 导 入 与 手 动 配 置
Import & Manual Config

推荐：一键导入

从管理员处获取个人链接

只接收发给你本人的 {MEMBER_CONFIG_URL}。不要要求管理员把所有人的链接发群里。

客户端选择“从剪贴板导入”

复制个人链接，进入客户端，选择从剪贴板/URL 导入。导入后节点名应显示为 {VPN_NAME}-{MEMBER_ID} 或管理员指定名称。

启用客户端代理或 VPN/TUN 接管

桌面系统一次只开启一个代理或 TUN 接管入口；手机系统启用 VPN 接管。ArchIToken-VPN 托盘只保留 全局代理 与 国内直连 两个模式，日常优先使用 国内直连。

验证出口

访问 https://ip.sb 或 https://ifconfig.me。看到管理员告知的 {PRIMARY_EXIT_IP} 或 {SECONDARY_EXIT_IP} 即成功。

托盘状态灯

绿色表示正常运行或当前出口，黄色表示可切换、待检测或预警，红色表示服务异常、节点不可用或出口异常。看到红色时不要重复扫码，先把错误截图按脱敏规则发给管理员。

手动配置字段模板

只有当客户端无法导入 URL 时才手动填写。以下字段必须由管理员单独给本人，不得从其他用户处复制。

字段模板说明
协议VLESS必须一致
地址{PRIMARY_SERVER_IP} / {SECONDARY_SERVER_IP}由管理员提供；先导入订阅，手动配置仅排障使用
端口{PORT}通常是 HTTPS 标准端口，但以管理员给出的值为准
UUID / ID{MEMBER_UUID}个人专属，不得共享
传输tcp以管理员提供为准
安全reality大小写按客户端要求填写
SNI / serverName{SNI}由管理员提供
PublicKey{REALITY_PUBLIC_KEY}不是私钥，但仍属于凭证组成部分
ShortID{REALITY_SHORT_ID}成员配置的一部分
Fingerprint{FINGERPRINT}常见值如 chrome；以管理员提供为准
Flow / Encryption{FLOW_OR_ENCRYPTION}客户端不支持时不要硬填，先换兼容客户端
04
Linux / WSL / Server CLI
Headless Setup

只给需要命令行环境的用户使用。普通桌面和手机用户可以跳过本节。

# 1) 仅示例：创建本机代理端口。不要把真实 UUID/密钥写入公开脚本。 sudo install -d -m 700 /etc/xray-client sudo nano /etc/xray-client/config.json # 2) 服务只监听本机，防止局域网泄露 # SOCKS: 127.0.0.1:10808 # HTTP : 127.0.0.1:10809 # 3) 校验并启动 /usr/local/bin/xray run -test -c /etc/xray-client/config.json sudo systemctl restart xray-client # 4) 验证 curl --socks5-hostname 127.0.0.1:10808 -4 https://ifconfig.me curl -x http://127.0.0.1:10809 -4 https://ifconfig.me
本机监听限制

CLI 代理端口必须只监听 127.0.0.1。不要监听 0.0.0.0，否则局域网内其他设备可能无密码使用你的代理。

05
团队发放、回收与换证
Credential Lifecycle
场景管理员动作用户动作
新成员加入创建个人 UUID / 链接，设置备注和限速，单独发放导入个人链接并验证出口
换设备可复用同一成员凭证或为新设备单独建凭证旧设备确认删除配置
设备丢失立即禁用该成员 UUID，必要时轮换 ShortID说明最后使用时间和设备类型
成员离职撤销个人凭证，清理备注和统计，更新台账删除本地客户端配置
疑似外泄先禁用个人凭证；若无法定位，轮换全部成员凭证等待管理员重新发新链接
不要单链接共享

单链接共享虽然省事，但一旦外泄无法定位责任人。团队使用应默认“一人一证”，这是隐私和可回收性的底线。

06
验收、故障排查与截图规则
Validation & Support

成功标准

检查项成功现象失败时处理
客户端已连接客户端状态显示 connected / running重新导入链接，确认没有复制断行
出口 IP测试页显示 {PRIMARY_EXIT_IP} 或 {SECONDARY_EXIT_IP}确认客户端代理、VPN 或 TUN 接管已开启
Google / GitHubHTTP 200 或正常打开换客户端或联系管理员查看服务端状态
某站 403说明“到达但被目标站拒绝”优先用浏览器测试，不把 403 当成 VPN 故障

截图求助前必须脱敏

  • 遮住 URL、二维码、UUID、节点链接、账号邮箱、浏览器左侧个人会话列表。
  • 只保留错误码、客户端名称、系统平台、连接状态、出口 IP 是否匹配。
  • 不要把完整配置 JSON、完整 VLESS URL、二维码截图发群里。
安全求助模板

“平台：Windows / 客户端：v2rayN / 状态：已连接 / 出口 IP：不匹配 / 错误：HTTP 403 或超时 / 已尝试：重新导入链接、重启客户端。”
