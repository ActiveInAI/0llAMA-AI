00
管理员安全基线
Security Baseline
核心原则

管理员不能把真实节点凭证写入任何“可群发文档”。所有 HTML、Markdown、PPT、截图、工单、AI 对话都视为可能被转发。真实凭证只存在于服务端配置、密码管理器、成员专属加密消息中；当前生产覆盖主/备两个出口节点。

当前生产节点清单

角色出口 / 服务器 IP用途发放规则
USA-LAX-A1 · 美国 洛杉矶203.0.113.10主节点、二维码主链接、首选出口验证美国 VPS 续费或流量恢复后继续作为主出口，禁止复用 Owner 凭证
NLD-AMS-A1 · 荷兰 阿姆斯特丹198.51.100.20备用节点、漫游切换、主节点维护兜底443 已验证可连接；主节点维护期间可优先发放备用入口
用户版处理

用户安全版文档仍使用占位符；真实 VLESS、订阅 URL、二维码和 UUID 只通过成员专属加密渠道发放。

01
成员凭证生命周期
Credential Lifecycle

默认策略：一人一证，一设备一备注

角色备注命名限速处理方式
Ownerowner-main / owner-dgx不限或高限额仅本人设备，绝不发给用户
成员member-a-laptop / member-a-phone按需限速每人独立 UUID，便于撤销
临时协作temp-yyyymmdd-project低限额 + 到期设置到期时间，到期自动失效
测试test-short-lived低限额测试完成立即删除
不要共享 Owner

Owner 凭证一旦泄露，所有流量都混在一起，无法定位责任人。团队越小越容易因为信任而偷懒，但运维上必须可撤销、可追踪、可审计。

发放 SOP

创建成员条目

在面板或服务端配置中创建新 UUID，备注使用内部代号，不使用真实姓名、手机号、邮箱。

生成个人链接

确认链接只包含该成员 UUID；不要复用 Owner 链接。截图二维码前先确认屏幕无其他凭证。

单独发送

通过密码管理器、端到端加密私聊或一次性安全渠道发给本人。禁止群发。

登记台账

记录成员代号、设备类型、发放时间、到期时间、限速、是否已验证。

02
文档防外泄脱敏检查
Redaction Gate

每次把 HTML/Markdown/PDF 发给用户前，先运行脱敏扫描。若命中 VPN URL、UUID、私钥、公钥、非批准真实 IP、面板路径、Webhook Token，就不允许发出。管理员版允许出现当前生产节点 IP，但用户版不允许出现完整凭证。

cat > /tmp/architoken-vpn-redaction-scan.py <<'PY' import re, sys, pathlib # 允许出现的文档/本机地址：环回、私网、RFC 5737 文档地址、占位符 ALLOW_IP_PREFIX = ( "127.", "10.", "192.168.", "172.16.", "172.17.", "172.18.", "172.19.", "172.20.", "203.0.113.", "198.51.100.", "192.0.2.", "0.0.0.0", "203.0.113.10", "198.51.100.20" ) PATTERNS = { "VPN URL": r"\b(vless|vmess|trojan|ss|hysteria2|tuic)://[^\s\"'<]+", "UUID": r"\b[0-9a-fA-F]{8}-[0-9a-fA-F]{4}-[0-9a-fA-F]{4}-[0-9a-fA-F]{4}-[0-9a-fA-F]{12}\b", "PrivateKey field": r"(?i)(privateKey|private_key|PrivateKey)\s*[:=]\s*[\"']?[^\"'\s<>{}]+", "PublicKey field": r"(?i)(publicKey|public_key|PublicKey|pbk)\s*[:=]\s*[\"']?[^\"'\s<>{}]+", "ShortID field": r"(?i)(shortId|shortIds|sid)\s*[:=]\s*[\"']?[^\"'\s<>{}]+", "Password/Token": r"(?i)(password|passwd|token|secret|sckey|sendkey|webhook)\s*[:=]\s*[\"']?[^\"'\s<>{}]+", "Panel path": r"https?://[^\"'\s<]+:[0-9]{2,5}/[A-Za-z0-9_-]{8,}", "Email": r"\b[A-Za-z0-9._%+-]+@[A-Za-z0-9.-]+\.[A-Za-z]{2,}\b", } ip_pat = re.compile(r"\b(?:\d{1,3}\.){3}\d{1,3}\b") failed = False for fname in sys.argv[1:]: p = pathlib.Path(fname) text = p.read_text(encoding="utf-8", errors="ignore") print(f"\n== {p} ==") for name, pat in PATTERNS.items(): matches = list(re.finditer(pat, text)) if matches: failed = True print(f"[FAIL] {name}: {len(matches)} hit(s)") for m in matches[:5]: s = m.group(0) print(" -", s[:120].replace("\n", "\\n")) ips = [] for m in ip_pat.finditer(text): ip = m.group(0) if not ip.startswith(ALLOW_IP_PREFIX): ips.append(ip) if ips: failed = True print(f"[FAIL] Public IPv4-like values: {len(set(ips))} unique") for ip in sorted(set(ips))[:10]: print(" -", ip) if not failed: print("[OK] no obvious secrets detected") sys.exit(1 if failed else 0) PY python3 /tmp/architoken-doc-redaction-scan.py \ "ArchIToken-VPN-团队安全接入指南.html" \ "ArchIToken-VPN-安全运维与隐私保护指南.html"
CI 建议

把扫描脚本放进内部仓库的 pre-commit 或 CI。任何含 vless://、UUID、真实公网 IP 的文档提交都直接失败。

03
日志最小化与自动清理
Log Minimization

VPN 运维的隐私重点是：不记录不必要的访问明细，保留短周期系统日志用于排障，自动清理临时文件和参数文件。

建议日志策略

日志类型建议保留周期
Xray access log默认关闭或只在排障窗口临时开启0-24 小时
Xray error log保留 warning/error7-30 天
systemd journal只保留服务状态和错误7 天
面板操作日志保留管理员操作审计30 天
参数文件使用后删除本机临时副本立即删除
# journal 只保留最近 7 天 sudo journalctl --vacuum-time=7d # logrotate 示例：Xray error/access 日志 sudo tee /etc/logrotate.d/xray >/dev/null <<'EOF' /var/log/xray/*.log { weekly rotate 4 compress delaycompress missingok notifempty create 0600 root root postrotate systemctl reload xray 2>/dev/null || systemctl restart xray 2>/dev/null || true endscript } EOF sudo logrotate -d /etc/logrotate.d/xray # 删除本机临时成员参数文件 shred -u /tmp/architoken-vpn-*.env 2>/dev/null || rm -f /tmp/architoken-vpn-*.env
排障窗口

需要打开 access log 时，先声明窗口，例如“打开 20 分钟排障”，排障后关闭并清理。不要长期记录用户访问目标域名。

04
监控告警与健康检查
Monitoring

监控只看服务是否可用，不采集用户访问内容。所有监控目标都用占位符，避免在文档中暴露真实面板路径。

监控项类型目标模板隐私注意
VPN 端口TCP{PRIMARY_SERVER_IP}:{VPN_PORT} / {SECONDARY_SERVER_IP}:{VPN_PORT}只探活，不记录用户流量
出口验证HTTP via proxyhttps://ifconfig.me按当前选中节点比对出口 IP：USA-LAX-A1=203.0.113.10，NLD-AMS-A1=198.51.100.20
面板 HTTPSHTTPS{PANEL_URL}仅管理员监控，不给用户
磁盘空间SSH / Agentdf -h /防日志占满
# 本机 xray-client 动态状态检查：期望出口取 config.json 第一条 outbound cat > /tmp/architoken-vpn-health.sh <<'EOF' #!/usr/bin/env bash set -euo pipefail expected="$(python3 - <<'PY' import json with open("/etc/xray-client/config.json", "r", encoding="utf-8") as f: c = json.load(f) for o in c.get("outbounds", []): v = o.get("settings", {}).get("vnext", [{}])[0] if v.get("address"): print(v["address"]) break PY )" actual="$(env -u http_proxy -u https_proxy -u all_proxy \ timeout 15 curl -fsS --socks5-hostname 127.0.0.1:10808 -4 https://ifconfig.me || true)" echo "expected=$expected" echo "actual=$actual" if [ "$actual" = "$expected" ]; then echo "OK" else echo "FAIL" exit 1 fi EOF chmod +x /tmp/architoken-vpn-health.sh /tmp/architoken-vpn-health.sh
05
泄露应急和回滚
Incident Response
疑似外泄时先止血

先撤销或禁用疑似成员凭证，再排查原因。不要先在群里讨论真实链接；不要把疑似泄露链接转发给更多人核对。

定位范围

通过成员代号、最近连接时间、流量异常和反馈来源判断是个人凭证外泄、HTML 文档外泄，还是管理员凭证外泄。

撤销个人凭证

删除或禁用对应 UUID。若是临时链接，直接到期或删除。通知该成员重新领证。

轮换公共参数

如果无法定位泄露者，轮换 ShortID / Reality 密钥 / 订阅路径，并为所有成员重新发个人链接。

清理文档

把所有旧版 HTML、截图、压缩包从群文件、网盘、工单和公开仓库删除。重新发布脱敏版。

复盘

记录外泄原因、影响范围、处理时间、是否需要更换 VPS 或 IP。更新发放 SOP。

配置回滚规范

# 创建当前正确配置的黄金备份，名字必须清晰 sudo cp -a /etc/xray-client/config.json \ /etc/xray-client/config.json.good.current.$(date +%Y%m%d_%H%M%S) # 回滚时只使用明确标注 good/current 的备份 sudo cp -a /etc/xray-client/config.json.good.current.YYYYMMDD_HHMMSS \ /etc/xray-client/config.json sudo systemctl restart xray-client
避免误回滚

不要使用含义不清的 config.json.bak.* 当作“当前正确配置”。备份文件名必须写明用途，例如 good.proxy-ams、before-rotation、old-line。

06
全平台支持台账
Support Matrix

给用户排障时，不要索要完整链接；只索要平台、客户端、是否导入成功、出口是否匹配、错误码。

平台一线检查常见问题处理
Windows系统代理或 TUN 接管是否被其他软件覆盖Clash / 浏览器插件冲突只保留一个代理或 TUN 接管入口
macOS网络权限和 VPN/TUN 接管状态二维码导入缺字段改为剪贴板导入
Linux服务是否监听 127.0.0.1:10808/10809systemd 未重启重启服务并看 journal
WSL2是否走 Windows 主机代理WSL 解析 localhost 不一致使用宿主机网关或 Windows 代理
AndroidVPN 权限是否授予后台被系统杀掉关闭电池优化
iOS客户端是否支持配置字段内核不兼容换兼容客户端或兼容配置
ChromeOSAndroid VPN 权限企业策略禁止 VPN联系设备管理员
NAS/Router插件是否支持 Reality把个人链接装到路由器导致全家共享单独给路由器建低权限凭证
ArchIToken-VPN 托盘基线

桌面托盘只展示 全局代理 与 国内直连 两个模式。状态灯采用交通灯语义：绿色正常，黄色预警或可切换，红色异常。系统代理仅作为命令行高级兜底，不作为日常模式发放。

安全支持模板

“请只发：平台、客户端名称、是否显示 connected、测试页出口 IP 是否等于管理员告知值、错误码。不要发完整配置链接、二维码或 JSON。”
